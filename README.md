# yadro_test_task1_DADPS
Distributed accelerometer data processing system in C++ (Linux)

# Распределённая система обработки данных акселерометра

## 1. Архитектура и взаимодействие компонентов

Система состоит из трёх узлов:

- **Узел A (отправитель)** — генерирует эмулированные данные акселерометра (x, y, z, timestamp) с частотой 50 Гц и отправляет их на сервер через TCP-сокет (порт 3425). 
- Все сообщения содержат поле `version` (целое число). Текущая версия — `1`.

**Сообщение от A к серверу:**
{"version": 1, "x": 1.23, "y": 4.56, "z": 7.89, "timestamp": 123456789}

- **Сервер (Linux)** — принимает данные от A, отбрасывает дубликаты (сравнение с последним пакетом с точностью до 1e-4), пересылает данные узлу B (порт 3426), принимает результат от B и отправляет его обратно узлу A.
- **Узел B (обработчик)** — получает от сервера данные (x, y, z, timestamp), вычисляет модуль ускорения `sqrt(x²+y²+z²)` и отправляет результат (timestamp, module) обратно серверу.

Все взаимодействия реализованы через **TCP-сокеты (IPv4)**. Для сериализации данных используется **JSON** (разделитель сообщений — символ `\n`). 

- Использован RAII-класс `tcp_server`: сокеты автоматически закрываются в деструкторе.
- Класс инкапсулирует все сетевые операции (`bind`, `listen`, `accept`, `send`, `recv`).
- Соблюдены принципы ООП.


Выбран JSON, потому что:

В задании явно указан JSON.

Упрощает отладку (можно использовать telnet или nc для ручной отправки сообщений).

Не требует компиляции схем (в отличие от Protobuf, Cap'n Proto).

Библиотека nlohmann/json — header-only, не требует дополнительной сборки.

Для небольшого объёма данных (1–10 сообщений в секунду) разница в размере и скорости не критична.


В рамках выполнения задания выбран вариант эмуляции данных на Linux, поэтому проблемы SELinux не возникали.

### Сборка с помощью CMake

```
# Клонирование репозитория
git clone <...>
cd <...>

# Создание директории для сборки
mkdir build && cd build

# Генерация файлов сборки
cmake ..

# Компиляция
make
```
После сборки исполняемые файлы server, client_a, client_b появятся в директории build/.
